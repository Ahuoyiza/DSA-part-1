Lesson 09

Stacks

"A stack is a Last-In, First-Out (LIFO) structure — like a stack of plates. You can only add or remove from the top. Both operations are O(1)."

LIFO — Last In, First Out

A stack enforces a strict access rule: the last item pushed is the first item popped. Think of a stack of plates — you can only take from the top. This constraint is actually a superpower: it makes stacks perfect for tracking "what came most recently."

Analogy

A stack of plates at a restaurant. The chef places clean plates on top (push). Servers always take from the top (pop). A plate on the bottom might wait a long time — it was the first in, but last out.

The Call Stack — Stacks in Your Runtime

Every time your code calls a function, the runtime pushes a stack frame onto the call stack. When the function returns, its frame is popped. This is why recursive functions can cause a "stack overflow" — too many nested calls exhaust the stack's fixed memory.

Watch out

Infinite recursion without a base case will overflow the call stack. JavaScript throws RangeError: Maximum call stack size exceeded. Always define your base case first — it's the equivalent of the null at the bottom of the stack.

Implementing a Stack

JavaScript 
// Stack backed by an array — all ops O(1)
class Stack {
  constructor() { this.data = []; }

  push(item) { this.data.push(item); }          // O(1)
  pop()       { return this.data.pop(); }        // O(1)
  peek()      { return this.data.at(-1); }      // O(1)
  isEmpty()   { return this.data.length === 0; }
  get size()  { return this.data.length; }
}

// ── Classic use: balanced brackets ─────────────────
function isBalanced(s) {
  const stack = new Stack();
  const pairs = { ')':'(', ']':'[', '}':'{' };

  for (const ch of s) {
    if ('([{'.includes(ch))  { stack.push(ch); }
    else if (ch in pairs) {
      if (stack.pop() !== pairs[ch]) return false;
    }
  }
  return stack.isEmpty();
}

isBalanced('({[]})');  // true
isBalanced('([)');     // false
TypeScript 
class Stack<T> {
  private data: T[] = [];

  push(item: T): void       { this.data.push(item); }
  pop(): T | undefined    { return this.data.pop(); }
  peek(): T | undefined   { return this.data.at(-1); }
  isEmpty(): boolean      { return this.data.length === 0; }
  get size(): number      { return this.data.length; }
}

// Evaluate postfix expression — O(n)
function evalPostfix(tokens: string[]): number {
  const stack = new Stack<number>();
  for (const tok of tokens) {
    if (!isNaN(Number(tok))) {
      stack.push(Number(tok));
    } else {
      const b = stack.pop()!;
      const a = stack.pop()!;
      if      (tok === '+') stack.push(a + b);
      else if (tok === '-') stack.push(a - b);
      else if (tok === '*') stack.push(a * b);
      else                   stack.push(a / b);
    }
  }
  return stack.pop()!;
}
// ["3","4","+","2","*"] → (3+4)*2 = 14

# DFS with an explicit stack (avoids recursion limit)
def dfs_iterative(graph, start):
    visited = set()
    stack   = [start]
    while stack:
        node = stack.pop()
        if node in visited:
            continue
        visited.add(node)
        for neighbour in graph[node]:
            if neighbour not in visited:
                stack.append(neighbour)
    return visited

# Balanced parentheses — O(n)
def is_balanced(s):
    stack, pairs = [], {')':'(', ']':'[', '}':'{'}
    for ch in s:
        if ch in '([{': stack.append(ch)
        elif ch in pairs:
            if not stack or stack.pop() != pairs[ch]:
                return False
    return not stack

Where Stacks Show Up

Function call stack

Every language runtime uses a stack to track function calls, local variables, and return addresses. It's why recursion works — and why it breaks when too deep.

Ctrl+Z (Undo)

Every keystroke is pushed onto an undo stack. Ctrl+Z pops the last action and reverses it. Ctrl+Y pushes it onto a redo stack — two stacks, infinite undo/redo.

Git stash

git stash is literally a stack. Each git stash push saves your current changes on top. git stash pop restores the most recent — LIFO behaviour built into version control.

Expression parsers

Compilers and calculators use stacks to parse and evaluate expressions — matching brackets, handling operator precedence, and converting infix to postfix notation.
