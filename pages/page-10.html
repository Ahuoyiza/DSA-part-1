Queues

"A queue is a First-In, First-Out (FIFO) structure — like a line at a coffee shop. Items enter at the back and leave from the front. Enqueue and dequeue are both O(1)."

FIFO — First In, First Out

A queue is the opposite of a stack. Items join at the back (enqueue) and leave from the front (dequeue). The first item added is always the first served — fair ordering is the queue's defining property.

Analogy

A queue is the coffee shop line. The first person in line is served first. A new customer joins at the back. Nobody cuts in — that's what makes it fair and predictable.

Queue vs Stack — Same Tools, Opposite Order

Queues and stacks both support O(1) insert and remove — but serve completely different ordering guarantees. The choice between them determines whether you process things in arrival order (queue) or reverse order (stack).

Queue (FIFO)
  • First in = first out
  • Enqueue at back, dequeue from front
  • Fair ordering — arrival order preserved
  • BFS traversal, task schedulers, print queues
  • Python: collections.deque
Stack (LIFO)
  • Last in = first out
  • Push and pop from same end (top)
  • Reverse ordering — most recent first
  • DFS traversal, undo/redo, call stack
  • Python: plain list with append/pop

BFS — The Classic Queue Algorithm

Breadth-First Search explores a graph level by level — all nodes at distance 1, then distance 2, and so on. A queue is the natural fit: enqueue neighbours, process them in arrival order. This guarantees the shortest path in an unweighted graph.

Key insight

BFS with a queue guarantees shortest path in an unweighted graph because it exhausts all paths of length k before exploring length k+1. Swap the queue for a stack and you get DFS — which explores as deep as possible first.

Implementing a Queue

JavaScript 
// Naive array queue — dequeue is O(n) due to shift()
class NaiveQueue {
  constructor() { this.data = []; }
  enqueue(item) { this.data.push(item); }    // O(1)
  dequeue()     { return this.data.shift(); } // O(n) ← problem!
}

// O(1) dequeue — use a pointer instead of shifting
class Queue {
  constructor() { this.data = []; this.head = 0; }

  enqueue(item) { this.data.push(item); }      // O(1)

  dequeue() {                                   // O(1)
    if (this.isEmpty()) return undefined;
    return this.data[this.head++];
  }

  peek()    { return this.data[this.head]; }
  isEmpty() { return this.head >= this.data.length; }
  get size(){ return this.data.length - this.head; }
}

// BFS using queue — O(V + E)
function bfs(graph, start) {
  const visited = new Set([start]);
  const queue   = new Queue();
  queue.enqueue(start);
  while (!queue.isEmpty()) {
    const node = queue.dequeue();
    for (const nb of graph[node] || []) {
      if (!visited.has(nb)) { visited.add(nb); queue.enqueue(nb); }
    }
  }
  return [...visited];
}
TypeScript 
class Queue<T> {
  private data: T[] = [];
  private head: number = 0;

  enqueue(item: T): void      { this.data.push(item); }

  dequeue(): T | undefined  {
    if (this.isEmpty()) return undefined;
    return this.data[this.head++];
  }

  peek(): T | undefined     { return this.data[this.head]; }
  isEmpty(): boolean        { return this.head >= this.data.length; }
  get size(): number        { return this.data.length - this.head; }
}

// Shortest path in unweighted graph — O(V + E)
function shortestPath(
  graph: Record<string, string[]>,
  start: string,
  end: string
): string[] | null {
  const queue = new Queue<string[]>();
  const visited = new Set([start]);
  queue.enqueue([start]);
  while (!queue.isEmpty()) {
    const path = queue.dequeue()!;
    const node = path[path.length - 1];
    if (node === end) return path;
    for (const nb of graph[node] ?? []) {
      if (!visited.has(nb)) {
        visited.add(nb);
        queue.enqueue([...path, nb]);
      }
    }
  }
  return null;
}
Python 
from collections import deque

# deque provides O(1) append (enqueue) and popleft (dequeue)
q = deque()
q.append('A')          # enqueue
q.append('B')
first = q.popleft()   # dequeue → 'A'  O(1)

# BFS — shortest path in unweighted graph
def bfs(graph, start, end):
    visited = {start}
    queue   = deque([[start]])
    while queue:
        path = queue.popleft()
        node = path[-1]
        if node == end:
            return path
        for nb in graph.get(node, []):
            if nb not in visited:
                visited.add(nb)
                queue.append(path + [nb])
    return None

Where Queues Show Up

CPU task scheduler

Operating systems use queues (often priority queues) to schedule processes. Each CPU core processes tasks in order, ensuring no process is starved indefinitely.

Message queues

Kafka, RabbitMQ, and SQS are industrial-scale queues. Producers enqueue messages; consumers dequeue at their own pace — decoupling services and absorbing traffic spikes.

Network packets

Routers buffer incoming packets in queues. When bandwidth is limited, packets wait their turn — the Internet is largely built on FIFO queues at every hop.

Notification delivery

Push notifications (iOS APNs, FCM) flow through message queues. Each notification enqueues to ensure delivery order and retry logic without overwhelming the device.
