Lesson 06

Arrays

"An array is a contiguous block of memory storing elements of the same type — accessed in O(1) by index, but costly to resize or insert in the middle."

What is an Array?

An array stores items in consecutive memory slots. Because each slot is the same size, the computer can jump directly to any element using simple arithmetic: address = base + index × size. No searching required — that's why index access is always O(1).

Analogy

An array is like a numbered row of lockers. You know locker #47 is exactly 47 steps from the start — you don't check every locker to find it. That's what O(1) access feels like.

Insert & Delete — The Expensive Part

The price of contiguous memory: when you insert at position i, every element from i to the end must shift one slot right. In the worst case (inserting at index 0) that's n shifts — O(n).

Watch out

Arrays have a fixed capacity. When you "push" to a dynamic array (like JavaScript's) and it runs out of space, it secretly allocates a new array 2× the size and copies everything over — that copy is O(n).

Common Array Operations

JavaScript 
const arr = [14, 32, 7, 91, 55];

// O(1) — direct index access
const val = arr[2];                   // 7

// O(1) amortised — push to end
arr.push(99);                         // [14,32,7,91,55,99]

// O(n) — insert at position 1 (shifts right)
arr.splice(1, 0, 100);               // [14,100,32,7,91,55,99]

// O(n) — remove from middle
arr.splice(3, 1);                    // removes index 3

// O(n) — linear search
const idx = arr.indexOf(91);         // 3 (or -1 if not found)

// O(n log n) — sort
arr.sort((a, b) => a - b);           // ascending
TypeScript 
const arr: number[] = [14, 32, 7, 91, 55];

// O(1) — direct index access
const val: number = arr[2];            // 7

// O(1) amortised — push to end
arr.push(99);

// O(n) — insert at middle
arr.splice(1, 0, 100);

// Type-safe search with find
const found = arr.find((x) => x > 50); // first >50

// Typed tuple for fixed-length arrays
const pair: [string, number] = ['age', 30];
Python 
arr = [14, 32, 7, 91, 55]

# O(1) — index access
val = arr[2]                          # 7
last = arr[-1]                        # 55 (negative index)

# O(1) amortised — append to end
arr.append(99)

# O(n) — insert at position
arr.insert(1, 100)

# Slicing — O(k) where k = slice size
sub = arr[1:4]                        # [100, 32, 7]

# List comprehension — O(n)
evens = [x for x in arr if x % 2 == 0]

Where Arrays Show Up

Image pixels

Every image is a 2D array of pixels (RGBA values). GPU shaders iterate it in parallel — O(1) access per pixel is what makes real-time rendering possible.

Git history

Git stores commit lists as arrays (with linked pointers). Log traversal is O(n); accessing a specific commit by index in a reflog is O(1).

Database rows

SQL tables store rows in pages — essentially arrays on disk. Sequential scans are O(n); indexed lookups reduce it to O(log n) via B-tree.

Audio buffers

Audio samples are stored in Float32Arrays. Web Audio API's processing nodes read them frame-by-frame — contiguous layout ensures cache-friendly O(1) reads per sample.
