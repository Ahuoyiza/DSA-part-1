Lesson 08

Linked Lists

"A linked list is a chain of nodes where each node holds a value and a pointer to the next. Unlike arrays, nodes live anywhere in memory — giving O(1) insert/delete but O(n) access."

The Node — Building Block

Every linked list is made of nodes. Each node is a small object with two fields: a value (the data) and a next pointer (the memory address of the next node). The last node's next is null — that's the end of the list.

Analogy

A linked list is like a treasure hunt. Each clue (node) tells you the location of the next clue (next pointer). To find clue #5, you must follow clues 1→2→3→4 first. That's why access is O(n), not O(1).

Singly vs Doubly Linked

A singly linked list has one pointer per node (next). A doubly linked list adds a prev pointer — doubling memory per node, but enabling O(1) backward traversal and making deletion easier when you already have a reference to the node.

Singly Linked
  • One pointer per node (next)
  • Less memory overhead
  • Deletion requires previous node reference
  • Forward traversal only
  • Stack, simple queues
Doubly Linked
  • Two pointers per node (prev + next)
  • 2× memory per node
  • O(1) deletion with just a node reference
  • Forward and backward traversal
  • Deque, browser history, LRU cache

O(1) Insert — Why It's Fast

Inserting a new node between two existing nodes only requires re-wiring two pointers — no shifting of elements like arrays. The catch: you first need a pointer to the node before the insertion point, which takes O(n) to find by traversal.

Interview tip

When asked "insert a node in O(1)", always clarify: O(1) requires already having a pointer to the predecessor node. Finding that node first is O(n). The insert itself — the pointer rewiring — is O(1).

Implementing a Linked List

JavaScript 
class Node {
  constructor(val) {
    this.val  = val;
    this.next = null;
  }
}

class LinkedList {
  constructor() {
    this.head = null;
    this.size = 0;
  }

  // O(1) — prepend to front
  prepend(val) {
    const node = new Node(val);
    node.next  = this.head;
    this.head  = node;
    this.size++;
  }

  // O(n) — traverse and find
  find(val) {
    let curr = this.head;
    while (curr) {
      if (curr.val === val) return curr;
      curr = curr.next;
    }
    return null;
  }

  // O(n) — delete by value
  delete(val) {
    if (!this.head) return;
    if (this.head.val === val) {
      this.head = this.head.next;
      this.size--;
      return;
    }
    let prev = this.head;
    while (prev.next) {
      if (prev.next.val === val) {
        prev.next = prev.next.next;  // skip over it
        this.size--;
        return;
      }
      prev = prev.next;
    }
  }
}
TypeScript 
class ListNode<T> {
  val:  T;
  next: ListNode<T> | null = null;
  constructor(val: T) { this.val = val; }
}

class LinkedList<T> {
  private head: ListNode<T> | null = null;
  private size: number = 0;

  prepend(val: T): void {
    const node = new ListNode(val);
    node.next  = this.head;
    this.head  = node;
    this.size++;
  }

  toArray(): T[] {
    const result: T[] = [];
    let curr = this.head;
    while (curr) {
      result.push(curr.val);
      curr = curr.next;
    }
    return result;
  }

  // Reverse in-place — O(n) time, O(1) space
  reverse(): void {
    let prev: ListNode<T> | null = null;
    let curr = this.head;
    while (curr) {
      const next = curr.next;
      curr.next  = prev;
      prev       = curr;
      curr       = next;
    }
    this.head = prev;
  }
}
Python 
class Node:
    def __init__(self, val):
        self.val  = val
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    # O(1) prepend
    def prepend(self, val):
        node      = Node(val)
        node.next = self.head
        self.head = node

    # O(n) traversal
    def to_list(self):
        result, curr = [], self.head
        while curr:
            result.append(curr.val)
            curr = curr.next
        return result

    # O(n) — detect cycle (Floyd's algorithm)
    def has_cycle(self):
        slow = fast = self.head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow is fast:
                return True
        return False

Access by indexO(1)O(n)Array

* Insert at middle requires finding the predecessor first (O(n) traversal). Insert marked O(1) assumes pointer already held. Array end-insert is amortised O(1).

Real-World Uses

Browser history

Each page visit is a node in a doubly linked list. Back/forward buttons are O(1) pointer moves — no shifting, no copying.

Text editor undo

Undo/redo stacks are often linked lists of "change records." Each undo is an O(1) pointer follow — the prior state doesn't need to be recomputed.

OS memory allocator

Free memory blocks are tracked in a linked list. When you call malloc, the allocator finds a suitable block in O(n) and unlinks it in O(1).

LRU Cache

A doubly linked list + hash map implements an LRU cache with O(1) get and put. The list tracks recency; the map gives O(1) node access for relinking.
