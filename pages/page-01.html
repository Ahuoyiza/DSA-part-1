Lesson 01

What is DSA?

"DSA — Data Structures and Algorithms — is the study of how to organise data and the step-by-step methods we use to process it."

Analogy

Think of a supermarket. The shelves are a data structure — they organise products so you can find them quickly. Your shopping strategy is an algorithm — whether you zigzag randomly or follow the aisle layout systematically determines how efficiently you finish. Same goal, very different performance.

The Core Concepts

What is a Data Structure?

A data structure is a way of organising and storing data in memory so that it can be accessed and modified efficiently. It is not about the data itself — it is about the container and the rules of that container. An array stores items in a numbered sequence. A linked list chains items together with pointers. A hash map pairs keys to values for near-instant lookup. Each structure is a different promise about how the data will behave when you interact with it.

Choosing the right data structure is often the most important engineering decision in a program. Use the wrong one and a simple operation becomes needlessly expensive. Use the right one and complex problems can feel almost trivial.

What is an Algorithm?

An algorithm is a finite, ordered sequence of well-defined instructions that transforms an input into a desired output. The word comes from the name of the 9th-century Persian mathematician Muhammad ibn Musa al-Khwarizmi. Crucially, an algorithm must always terminate — it cannot run forever — and for the same input it must always produce the same output.

Algorithms are language-agnostic. The same binary search algorithm can be implemented in Python, JavaScript, Rust, or written in pseudocode. The language is a detail. The logic is the algorithm.

Why They Are Inseparable

Data structures and algorithms are two sides of the same coin. An algorithm operates on a data structure — and the choice of data structure determines which algorithms are efficient and which are not. You cannot analyse an algorithm in isolation without knowing what it is operating on. Binary search is O(log n) only on a sorted array. It becomes meaningless on a linked list. Understanding both together is what gives you real power as an engineer.

Four Qualities of a Good Algorithm

Correctness

It must produce the right output for every valid input, including edge cases like empty arrays, single elements, or duplicate values.

Efficiency

It should use the minimum necessary time and memory. An algorithm that works but takes 10 minutes is not useful in production.

Clarity

Another engineer should be able to read and understand it without a long explanation. Clever code that is hard to read is a liability.

Finiteness

It must terminate for any valid input. An algorithm that could run indefinitely is not an algorithm — it is an infinite loop waiting to happen.

Pseudocode

Pseudocode
// Find the maximum number in a list FUNCTION findMax(list): IF list is empty: RETURN null max ← list[0] // assume first element is the max FOR EACH element in list: IF element > max: max ← element // found a new maximum RETURN max // Time: O(n) — must check every element // Space: O(1) — only one extra variable (max)

Implementation

JavaScript 
/**
 * findMax — Returns the largest number in an array.
 * Time:  O(n) — single pass through every element
 * Space: O(1) — only a single variable regardless of input size
 */
function findMax(arr) {
  if (arr.length === 0) return null;   // guard: empty input

  let max = arr[0];                     // step 1: assume first is largest

  for (let i = 1; i < arr.length; i++) { // step 2: walk the array
    if (arr[i] > max) {               // step 3: update if bigger
      max = arr[i];
    }
  }

  return max;                          // step 4: return the winner
}

// ── Usage ──────────────────────────────────────────────
const scores  = [42, 7, 98, 31, 55, 6];
const highest = findMax(scores);
console.log(highest); // 98

console.log(findMax([]));         // null  — empty array
console.log(findMax([-5, -1, -8])); // -1   — all negative
console.log(findMax([3]));         // 3    — single element

/**
 * findMax — Returns the largest number in an array.
 * Time:  O(n) — single pass through every element
 * Space: O(1) — only a single variable regardless of input size
 */
function findMax(arr: number[]): number | null {
  if (arr.length === 0) return null;   // guard: empty input

  let max: number = arr[0];             // step 1: assume first is largest

  for (let i: number = 1; i < arr.length; i++) {
    if (arr[i] > max) {               // step 2: update if bigger
      max = arr[i];
    }
  }

  return max;
}

// ── Usage ──────────────────────────────────────────────
const scores: number[] = [42, 7, 98, 31, 55, 6];
const highest: number | null = findMax(scores);
console.log(highest); // 98

// TypeScript catches misuse at compile time:
// findMax(['a', 'b']); // Error: string[] not assignable to number[]

Complexity of findMax

Operation Time Space Why
Access arr[0] O(1) O(1) Direct index access, no traversal needed
Single comparison O(1) O(1) One operation, independent of input size
Full findMax pass O(n) O(1) Must visit every element once; only stores max
Best case (max is first) O(n) O(1) Still must verify no larger element exists later

Where You'll See This in the Real World

Google Search

Inverted index (data structure) + PageRank (algorithm) together retrieve the most relevant result from billions of pages in milliseconds.

GPS Routing

A graph (data structure) represents roads and junctions. Dijkstra's algorithm (algorithm) finds the shortest path between any two nodes.

Social Media Feed

A priority queue (data structure) holds posts ranked by engagement score. A ranking algorithm re-orders them each time you refresh.

Bank Transactions

A B-tree (data structure) indexes millions of records. A ledger reconciliation algorithm verifies balance consistency across distributed systems.
