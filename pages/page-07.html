Lesson 07

Strings

"A string is a sequence of characters — under the hood, an array of bytes. Immutability, encoding, and the hidden cost of concatenation define most string performance traps."

Strings Are Character Arrays

At the memory level, a string is just an array of character codes. "hello" is stored as five bytes: [104, 101, 108, 108, 111]. Every array operation from Lesson 06 applies — but most languages make strings immutable, meaning any "modification" creates a brand new string in memory.

Key insight

In Python and JavaScript, strings are immutable. Every + operator creates a brand new string. Concatenating n strings in a loop is O(n²) total — not O(n). Use array joins or string builders instead.

The Concatenation Trap

Each concatenation allocates a new string and copies both halves into it. Doing this n times means copying 1+2+3+…+n characters = O(n²) total work.

O(n²) — naive loop
  • String concat in a loop (s += word)
  • Each iteration copies all previous characters
  • Kills performance at n > 10,000
  • Common interview trap in Python/JS
O(n) — array join
  • Collect parts in array, call .join('') once
  • Single allocation for the final string
  • Python: ''.join(parts)
  • Java: StringBuilder.append()

String Patterns in Code

JavaScript 
// ── Common string operations ──────────────────────
const s = 'Hello, World!';

// O(1) — length, index access
s.length;                            // 13
s[0];                                // 'H'

// O(n) — includes, indexOf
s.includes('World');                // true
s.indexOf('o');                     // 4

// O(k) — slice, substring
s.slice(7, 12);                      // 'World'

// ── O(n²) trap ────────────────────────────────────
let result = '';
for (const word of words) {
  result += word + ' ';            // copies grow each time
}

// ── O(n) fix ──────────────────────────────────────
const result2 = words.join(' ');    // single allocation

// ── Reverse a string (interview classic) ──────────
const reversed = s.split('').reverse().join(''); // O(n)
TypeScript 
// Check if string is palindrome — O(n)
function isPalindrome(s: string): boolean {
  let left = 0, right = s.length - 1;
  while (left < right) {
    if (s[left] !== s[right]) return false;
    left++;
    right--;
  }
  return true;
}

// Count character frequency — O(n)
function charFreq(s: string): Map<string, number> {
  const freq = new Map<string, number>();
  for (const ch of s) {
    freq.set(ch, (freq.get(ch) ?? 0) + 1);
  }
  return freq;
}

// Anagram check — O(n)
function isAnagram(a: string, b: string): boolean {
  if (a.length !== b.length) return false;
  const freq = charFreq(a);
  for (const ch of b) {
    const count = freq.get(ch);
    if (!count) return false;
    freq.set(ch, count - 1);
  }
  return true;
}
Python 
s = "Hello, World!"

# O(1) — index access
s[0]                              # 'H'
s[-1]                             # '!'

# O(k) — slice
s[7:12]                           # 'World'

# O(n) — reverse
s[::-1]                           # '!dlroW ,olleH'

# O(n²) trap — avoid this
result = ""
for word in words:
    result += word                 # copies grow

# O(n) fix — join
result = " ".join(words)           # single alloc

# Counter for frequency map — O(n)
from collections import Counter
freq = Counter("banana")          # {'a':3,'n':2,'b':1}

# Palindrome check — O(n)
def is_palindrome(s):
    return s == s[::-1]

Where String Perf Matters

Search engines

Indexing and querying trillions of strings. Inverted indexes, prefix tries, and n-gram models all depend on efficient string operations to respond in milliseconds.

Security & hashing

Password hashing (bcrypt, Argon2) converts string inputs into fixed-length digests. Timing-safe comparison prevents leaking length information via early exit.

Code editors

VS Code uses a "piece tree" to represent file content as a series of string segments — avoiding O(n²) edits when you insert characters in large files.

DNS & routing

DNS lookups match domain strings using trie data structures. Prefix matching is O(m) where m is the domain length — independent of how many domains exist.
